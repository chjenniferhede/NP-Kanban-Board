# Kanban Backend API

TypeScript + Express + Drizzle ORM + Zod API server for a Kanban board application.

## Architecture

```
src/
├── index.ts              # Express server entry point
├── db/
│   ├── client.ts         # Drizzle client & connection setup
│   └── schema.ts         # Drizzle ORM table schemas
├── routes/
│   └── users.ts          # Example route with full CRUD + validation
├── validators/
│   └── index.ts          # Zod schemas for all request bodies & params
├── middleware/
│   └── validation.ts     # Validation middleware factories
└── types/
    └── index.ts          # Shared TypeScript types
```

## Setup

### 1. Environment Variables

Copy `.env.example` to `.env` and fill in your Supabase PostgreSQL credentials:

```bash
cp .env.example .env
```

```env
DATABASE_URL=postgresql://user:password@db.supabase.co:5432/postgres
PORT=3000
NODE_ENV=development
```

### 2. Install Dependencies

```bash
npm install
# or
pnpm install
```

### 3. Create & Run Migrations

Generate migration files from your schema:

```bash
npm run drizzle:generate
```

Then apply migrations:

```bash
npm run drizzle:migrate
```

## Development

Start the development server with hot reload:

```bash
npm run dev
```

Server runs at `http://localhost:3000`

## Building & Production

Build TypeScript:

```bash
npm run build
```

Start production server:

```bash
npm start
```

## API Patterns

### Validation with Zod

All routes validate inputs using Zod schemas. Define schemas in `src/validators/index.ts`:

```typescript
export const createUserSchema = z.object({
  email: z.string().email(),
  name: z.string().min(1).max(255),
});
```

### Using Validation Middleware

Apply validation to routes using middleware:

```typescript
import { validateBody } from "../middleware/validation.js";
import { createUserSchema } from "../validators/index.js";

router.post("/", validateBody(createUserSchema), async (req, res) => {
  // req.body is type-safe and validated
  const { email, name } = req.body;
  // ...
});
```

### Example Route

See `src/routes/users.ts` for a complete example with:

- Pagination (query params)
- CRUD operations
- Zod validation on body, params, and query
- Error handling
- Database queries with Drizzle

### Database Queries

Use the `db` instance from `src/db/client.ts`:

```typescript
import { db } from "../db/client.js";
import { users } from "../db/schema.js";

// Select
const allUsers = await db.select().from(users);

// Insert
const newUser = await db
  .insert(users)
  .values({ email: "user@example.com", name: "John" })
  .returning();

// Update
const updated = await db
  .update(users)
  .set({ name: "Jane" })
  .where(eq(users.id, userId))
  .returning();

// Delete
await db.delete(users).where(eq(users.id, userId));
```

## Adding New Routes

1. Create a new file in `src/routes/` (e.g., `boards.ts`)
2. Define Zod schemas for the route in `src/validators/index.ts`
3. Implement routes with validation middleware
4. Import and register router in `src/index.ts`: `app.use('/api/boards', boardsRouter)`

## Database Schema

Define new tables in `src/db/schema.ts` using Drizzle:

```typescript
export const boards = pgTable("boards", {
  id: uuid("id")
    .primaryKey()
    .default(sql`gen_random_uuid()`),
  title: varchar("title", { length: 255 }).notNull(),
  // ...
});
```

Then:

1. Run `npm run drizzle:generate` to create migration file
2. Review the generated migration in `drizzle/`
3. Run `npm run drizzle:migrate` to apply it

## Error Handling

API returns JSON error responses with details:

```json
{
  "success": false,
  "error": "Validation failed",
  "details": [
    {
      "path": "email",
      "message": "Invalid email address"
    }
  ]
}
```

## Notes

- All IDs are UUIDs generated by PostgreSQL (`gen_random_uuid()`)
- Timestamps are automatically managed (`createdAt`, `updatedAt`)
- Zod validation happens before database operations
- Errors are caught and returned as JSON responses
